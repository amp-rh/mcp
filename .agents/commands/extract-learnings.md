# /extract-learnings

Document patterns and insights discovered during development.

## Usage

```
/extract-learnings
```

## Steps

1. Review recent work and identify patterns worth documenting
2. Check if pattern already exists in `.agents/learnings/`
3. Create new file or update existing one
4. Format as actionable guidance for future sessions

## Template

```markdown
# Learning: [Title]

## Context
What situation led to this learning?

## Pattern
What is the pattern or insight?

## Example
Show concrete example of applying this.

## Related
- Links to related docs or code
```
